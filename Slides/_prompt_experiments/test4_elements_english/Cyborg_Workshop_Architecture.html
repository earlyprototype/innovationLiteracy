02. Prompt Assembly Process

How the abstract human data is merged with strict system constraints to create an executable master prompt that prevents AI hallucination.
sequenceDiagram autonumber actor Participant participant Mural as Logic Map (Mural) participant UI as Component Library participant Style as Style Guide participant Prompt as Master Prompt participant AI as NotebookLM Participant->>Mural: Define 'Objects' & 'Verbs' Participant->>UI: Select allowable UI elements Participant->>Style: Retrieve visual constraints Note over Participant, Prompt: The Cognitive Load Participant->>Prompt: Combine: Persona + Constraints + Logic Map Prompt->>AI: Submit structured instructions AI-->>Participant: Generate low-fi interactive artifact Note over Participant, AI: If output fails, the Participant fixes the Prompt/Map, NOT the code.

03. The "Holodeck" Architecture

The structural mapping of the NotebookLM environment. Showing participants that the AI is an empty room until they define the physics (Sources).
