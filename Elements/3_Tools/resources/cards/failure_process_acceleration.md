# Failure Mode 6: Process Acceleration Without Integrity

## What Happens

AI speed tempts teams to skip essential process steps. Process collapses to "problem brief → AI solution → ship."

## Why It Occurs

- Organisational pressure for speed
- AI provides excuse to cut corners
- Outputs look complete even when thinking isn't
- Misunderstanding of where AI adds value

## How It Shows Up

- **Skipping Discovery:** "We don't need user interviews, AI can tell us user needs"
- **Skipping Iteration:** "First AI output looks good, ship it"
- **Skipping Testing:** "AI validated our idea" (echo chamber effect)

## Example

**Collapsed Process:**
1. Product brief: "Users need better analytics"
2. Prompt AI: "Design analytics dashboard"
3. AI generates comprehensive UI
4. Deploy to production

**Missing:** Discovery (what analytics? for what decisions?), Define (what problem?), Ideate (multiple approaches?), Prototype (low-fi validation?), Test (do users use it?)

**Result:** Feature that looks complete but doesn't solve actual problem.

## Prevention

1. **Maintain process gates:** AI accelerates WITHIN phases, doesn't skip phases
2. **Position AI as accelerator not replacement:** "AI helps us synthesise faster so we can iterate MORE" — speed benefit is more cycles, not fewer steps
3. **Facilitation discipline:** Hold the process structure. Don't let pressure collapse stages

## When It Matters Most

**Under time pressure** — precisely when process integrity most threatened and most valuable.
