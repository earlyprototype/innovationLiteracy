# Facilitation: Pre-Session Preparation

## Tool Selection & Setup

**Match AI tool to task:**
- **NotebookLM:** Research synthesis, document analysis (source-grounded)
- **ChatGPT/Claude:** Ideation expansion, reframing
- **Perplexity:** Cited research, background context
- **Mural AI:** Visual clustering, organisation
- **v0/Bolt.new:** Rapid functional prototyping (later stages only)

**Pre-test checklist:**
- [ ] Run through planned AI interactions before session
- [ ] Understand output patterns and quality thresholds
- [ ] Prepare fallback plans for unusable AI output
- [ ] Test time requirements: How long does synthesis actually take?

## Control Model Decision

| Model | Pros | Cons | When to Use |
|-------|------|------|-------------|
| Facilitator-controlled | Faster, less chaos | Less ownership | Demos, time pressure |
| Participant-controlled | Builds capability | Slower, more support needed | Preferred — co-creation |

**Recommendation:** Participant-controlled where possible.

## Time Budgeting

**Critical: Budget MORE time for AI-augmented tasks, not less.**

- Manual synthesis: 15 min pattern identification
- AI-augmented: 5 min generation + 15 min interpretation = **20 min total**

**Why longer:** Reading AI output, verification against sources, group discussion of interpretation, prompt iteration when first output inadequate.

**Where speed gains occur:** Documentation grunt work, generating variations, structuring messy data, post-workshop synthesis.

## Framing Language (Prepare in Advance)

- **"AI is rendering engine for YOUR logic, not solution generator"**
- **"Bad AI output is useful diagnostic data"**
- **"We're teaching two things simultaneously"** — innovation process AND AI-augmented practice

**For failure moments:**
- "This nonsense is useful — what did we fail to specify?"
- "Let's diagnose this together"
