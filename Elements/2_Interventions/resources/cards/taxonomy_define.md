# Interventions Taxonomy: Define Stage

## Purpose

Synthesise discovery findings into clear, actionable problem statement.

## Common Failure Modes

- **Problem/solution confusion:** Framing solution as problem
- **Too broad:** Everything is the problem
- **Too narrow:** Missing system context
- **Stakeholder misalignment:** Different people solving different problems

## Pattern Interrupts

- **"Reframe that as a question"** — linguistic forcing function
- **"Remove your solution from that problem statement"** — catch solution-jumping
- **"What's the problem behind that problem?"** — move upstream

## Structured Methods

- How Might We statements (problem reframing as possibility)
- Point of View statements (user + need + insight)
- Problem space mapping (system context visualisation)
- Five Whys (root cause identification)
- Assumption inventory (document what must be true)

## Technology Adoption Specific

- As-Is vs To-Be mapping (current → desired state)
- Adoption barrier analysis (why current tech persists despite pain)
- Change readiness assessment
- Value proposition framing (benefit articulation for different user groups)
