# Interventions Taxonomy: Prototype & Test Stages

## Prototype — Purpose

Make thinking tangible and testable. Prototypes are questions, not answers.

### Prototype Failure Modes

- **Over-production:** Building too complete, too early
- **Attachment:** Defending rather than testing
- **Perfectionism:** Aesthetic over function
- **Analysis paralysis:** Planning instead of making

### Prototype Pattern Interrupts

- **"Build it in 7 minutes"** — time constraint forces simplicity
- **"It should break"** — permission for rough
- **"What's the riskiest assumption?"** — focus on uncertainty
- **Material constraint:** "Use only paper and tape"

### Prototype Methods

- Fidelity spectrum (paper → digital → functional)
- Experience prototyping (simulate the interaction)
- Wizard of Oz (manual backend, polished frontend)
- Storyboarding (sequence the experience)

---

## Test — Purpose

Put prototypes in front of real users. Testing is investigation, not validation.

### Test Failure Modes

- **Confirmation bias:** Seeking validation not learning
- **Defensive posture:** Explaining rather than observing
- **Unstructured feedback:** No plan for what to learn
- **Missing documentation:** Insights lost immediately

### Test Pattern Interrupts

- **"Don't defend your prototype"** — observe, don't justify
- **"What did you notice?"** not "Did you like it?" — observation over opinion
- **"What surprised you?"** — catch unexpected responses
- **Silent observation protocols**

### Test Methods

- Feedback capture templates
- Think-aloud protocols
- A/B testing frameworks
- Success/failure criteria (define learning targets before testing)
- Iteration planning (what changes based on what feedback?)
