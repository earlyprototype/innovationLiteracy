# The Alternation Pattern

## The Rhythm

Design Thinking deliberately alternates between divergent and convergent modes:

| Stage | Diverge | Converge To |
|-------|---------|-------------|
| **Discovery** | Wide exploration of user needs, contexts, pain points | Key insights |
| **Define** | Multiple problem framings | Clear problem statement |
| **Ideate** | Wide range of solution concepts | Promising concepts |
| **Prototype** | Multiple approaches to same concept | Testable prototype |
| **Test** | Gather wide range of feedback | Learning and next actions |

The rhythm is: **open possibilities, then close to decision.** Each cycle narrows uncertainty and builds evidence base for better decisions.

## Why This Matters

**The mixing problem:** Teams that evaluate during ideation kill creativity. Teams that refuse to converge remain in perpetual exploration.

**The timing problem:** Operational culture is naturally convergent — it rewards decisiveness and closure. Innovation requires deliberate divergent phases *protected from convergent pressure*.

**The facilitation requirement:** Groups don't naturally alternate. Facilitation is essential to:
- Name which mode is active
- Interrupt mode violations
- Manage transitions
- Protect divergent time from operational gravity

## Common Mistakes

| Pattern | Symptom | Impact |
|---------|---------|--------|
| Convergent-only | "Let's just pick something" | Narrow framing, incremental solutions |
| Divergent-only | "We need more options first" | Perpetual ideation, no implementation |
| Unbalanced | 10min diverging, 50min converging | Shallow exploration, rushed decisions |
| Implicit | People diverging while others converge | Frustration, suboptimal outcomes |
