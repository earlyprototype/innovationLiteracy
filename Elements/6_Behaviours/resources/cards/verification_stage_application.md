# Verification Discipline: Stage Application

## Per-Stage Verification Focus

### Discovery (Empathise)
**Critical checks:** Source Grounding, Interpretation Ownership
- Trace every AI pattern back to specific interview quotes
- Flag patterns without source material support
- Check if AI smoothed contradictions in user feedback
- **Language:** "Can someone find the actual quote where this user mentioned X?"

### Define (Frame)
**Critical checks:** Alignment, Assumption Surfacing
- Does AI problem framing address YOUR challenge from discovery?
- Has it drifted to adjacent easier problem?
- Check for embedded solutions in problem statements
- **Language:** "This framing sounds good — does it reflect what users actually said?"

### Ideate (Generate)
**Critical checks:** Tension, Alignment to Problem
- Do AI ideas show trade-offs or only benefits?
- Are implementation complexities acknowledged?
- Do expanded ideas still address the defined user need?
- **Language:** "These ideas sound great — what downsides aren't we seeing?"

### Prototype (Build)
**Critical checks:** Alignment to Intent, Iteration Discipline
- Does AI artifact reflect YOUR logic?
- Are gaps in output revealing gaps in prompt specification?
- Did source documents actually constrain output?
- **Language:** "This isn't quite right — what did we underspecify?"

### Test (Validate)
**Critical checks:** Source Grounding, Contradiction Preservation
- Trace every AI theme to specific user quotes
- Check for false quantification ("most users") without data
- Verify edge cases and contradictions preserved
- **Language:** "AI says users struggled with X — which test sessions showed that?"

## Task-Specific Checklists

**Synthesis:**
- [ ] Trace each pattern to source quotes?
- [ ] Contradictions preserved?
- [ ] Confirming or challenging data?
- [ ] Maintained interpretation ownership?

**Ideation Expansion:**
- [ ] Ideas address defined problem?
- [ ] Trade-offs visible?
- [ ] Implementation complexity realistic?
- [ ] Evaluated by my criteria, not AI's?
