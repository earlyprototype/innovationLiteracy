# Verification Discipline: Team Integration

## Why Team Verification > Individual Verification

**Individual (weak):** Each person checks privately, failures go unnoticed by team.
**Shared (strong):** Team runs checks together, failures become learning moments.

## How to Build Shared Practice

### 1. Facilitator Models Visibly
- Talk through checks aloud: "Before we accept this, let's verify against sources"
- "I'm running an alignment check — did AI answer our question?"

### 2. Create Shared Language
- "Let's run a tension check on this"
- "Can someone do source grounding for this claim?"
- "I'm getting echo chamber vibes — is this challenging our assumptions?"

### 3. Celebrate Catches
- "Good catch — that claim isn't grounded"
- "Great alignment check — AI drifted to adjacent problem"

### 4. Document Decisions
- Capture which gates flagged concerns
- Note what was revised based on checks
- Make verification visible in shared workspaces

## Pair Verification Practice

When learning, practice in pairs:
1. One person generates prompt and output
2. Other runs Four Gates checks
3. Discuss findings together
4. Switch roles

Builds shared understanding faster than individual practice.

## Common Resistance & Responses

| Resistance | Response |
|-----------|----------|
| "This slows us down" | 30 seconds per check. Building on flawed outputs wastes hours |
| "I trust AI outputs" | Trust requires verification. Blind trust creates predictable failures |
| "AI outputs look perfect" | That's when verification is most needed |
| "We don't have time" | Time to verify now or time to fix failures later — choose |
